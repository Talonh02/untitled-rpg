Overview

What Makes It Different

Every AI game on the market uses one model for everything. Narrator, villain, shopkeeper, love interest — all the same voice. No real opposition, no consequences, no world that moves without you.

AI roleplay market: ~$500M-1.2B (2025), projected $8.9B by 2033. Nobody has multi-model adversarial architecture.

Overview

Core Philosophy

Models speak. Python thinks.

Every decision in this architecture follows one rule: AI models generate language, Python runs the simulation. No model ever decides if you live or die.

7 Design Principles

  1. Models do reasoning, Python does simulation
  2. The world moves without you — consequences compound
  3. Characters scale model tier with dramatic importance
  4. NPCs only know what they'd plausibly know
  5. The system is genre-agnostic — the player decides the story
  6. Real stakes — you can die, fail, lose, be betrayed
  7. Every playthrough is a new generated world

The World

World Generation Pipeline

Opus creates meaning. Python creates space.

Step 1: Opus

One big call (~$0.30-0.50). Returns 5,000-10,000 tokens of structured JSON: continents, regions, cities, factions, history, philosophy, 30-50 named characters with fate scores, an antagonist, intellectual traditions.

Step 2: Python

Perlin noise terrain. Cities placed on valid coordinates (trade ports on coasts, etc.). Roads generated. Travel times calculated. NPC stat blocks rolled for all unnamed characters. No model calls.

Step 3: Character Author

Opus writes system prompts for high-fate characters (30-50 calls). Batched during loading screen.

Step 4: Visual Assets

World map, NPC portraits, region art. ~30-40 image API calls batched while player reads lore text.

The World

The Director

One cheap model call per in-game day. Reads compressed world state. Outputs 0-5 structured events. This is the magic.

What The Director Does

  • Moves NPCs between locations
  • Advances faction plans (the army marches, the treaty falls apart)
  • Spreads consequences (the merchant you cheated is one town closer)
  • Spreads reputation (you arrive somewhere new, the bartender already looks nervous)
  • Generates ambient events (a festival, a plague, a shipwreck)

Characters

Fate vs Depth

These are independent axes. Fate = narrative importance. Depth = inner life richness. They don't correlate.

High Fate, High Depth The hidden queen. World-shaping AND philosophically rich. Opus prompt, 1500+ tokens. Rare.
Low Fate, High Depth The washerwoman who stops you cold. Rich inner life, no narrative importance. The moment that feels like literature.
High Fate, Low Depth The brave farm boy. Central to the plot, not especially interesting to talk to. Flash model, short prompts.
Low Fate, Low Depth Background scenery. The 40 people in the bar. Pure data, no model call. They're dockworkers arguing over dice.

Fate reshapes the stat distribution — pulling the mean upward and widening variance. A fate-touched person has higher ceiling AND more variance. Spikes and valleys.

Characters

Continuous Character Scaling

No hard tiers. Everything scales smoothly with the depth score.

depth_score = depth*0.30 + intelligence*0.20 + wisdom*0.15
            + empathy*0.15 + creativity*0.10 + education*0.10
prompt_tokens = 40 + (depth_score * 14)    // 110 to 1440 tokens
max_response_words = 15 + (depth_score * 1.5) // 30 to 165 words
Score 5: "Yeah." Score 40: Short paragraph Score 80: Makes you think Score 95: Complete human

Model Selection (Fuzzy Boundaries)

effective_score = depth_score + (fate * 10)
roll = effective_score + random.uniform(-8, 8)

if roll < 30:  flash_lite    ~55% of NPCs
elif roll < 52: flash         ~25%
elif roll < 72: sonnet        ~13%
elif roll < 88: sonnet_rich   ~5%
else:           opus          ~2%

A depth-28 NPC has a ~15% chance of rolling into Flash territory. Sometimes an ordinary person surprises you. The randomness IS the realism.

The Character Author

Opus reads the full stat block and writes a soul. Not a lookup table. It translates numbers into vibes.

Depth ~25 (110 tokens)

"Renn, dockworker. Speaks in short declaratives. Doesn't think much about why things are the way they are. Loyal to his crew. If you ask him about the war, he shrugs. Gets irritated if pressed."

Depth ~80 (1160 tokens)

"Dalla Cairn, former academy scholar turned herbalist. Speaks precisely but warmly — she chose simple words after years of jargon. Carries a guilt she won't name: her research was used to justify the northern purge. Notices emotional undercurrents others miss. Will test you with silence before deciding if you're worth her real thoughts. Deeply lonely but won't admit it."

Opus thinks in qualitative bands:

  • 0-15: Thinks in simple terms. Present-focused.
  • 30-50: Real personality. Might surprise you once.
  • 50-70: Genuinely thoughtful. Has examined their life.
  • 85-100: Changes how you think about something. A voice unlike anyone else.

Pure data. No model call. ~22 stats, randomly generated. The 40 people in a bar. Scenery.
↓ player talks to them

One model call generates a 200-500 token context prompt consistent with stats and local lore. Now they're a person.
↓ player walks away

Full backstory, secrets, contradictions. 800-1500 token Opus-authored prompts. Can become companion.

Companions can refuse orders, leave if trust drops, and die permanently. Their death is always narrated with weight.

Combat

Auto-Resolve, Not Turn-Based

The player doesn't click "attack" repeatedly. They decide to fight. Python resolves the entire encounter instantly. Sonnet writes the scene.

Player decides to fight
"I punch the bartender"
↓
Interpreter (Flash) parses intent
{"action": "combat", "target": "npc_bartender", "method": "punch"}
↓
Python Stat Engine resolves instantly
CPR calculation, weapon-armor matrix, numbers advantage, randomness
↓
Narrator (Sonnet) writes the scene
Narration length scales with drama. Decisive victory = 2 paragraphs. Companion death = full page.

Minimum calls per turn: 2 (interpreter + narrator). Combat is Python + Sonnet. No combat model needed.

Combat

The Combat Math

Combatant Power Rating (CPR)

CPR = (STR * 0.25 + AGI * 0.25 + TGH * 0.20
     + CRG * 0.10 + PER * 0.10 + WIL * 0.10)
     * weapon_mult * armor_mult

Weapon-Armor Matrix

NoneLeatherChainPlate
Unarmed1.00.60.30.15
Dagger1.00.80.40.2
Long sword1.00.950.70.5
2H sword1.01.00.80.6
Mace0.90.951.00.85
Bow1.00.850.50.3

Numbers Advantage (Log2 Scaling)

ratio = your_side / their_side
bonus = 1 + log2(ratio) * 0.4
// 2v1 = 1.4x, 3v1 = 1.63x, 5v1 = 1.93x

Final step: +/-20% randomness. Then margin = (player_roll - enemy_roll) / total.

Combat

Decision Points

When margin is within ±0.08, the game pauses and asks what you do.

Margin Close

"The fight could go either way. He has you pinned but is tiring. What do you do?"

Clever/tactical choice: +0.10 to +0.15 margin. Reckless: random(-0.10, +0.15). Passive/pleading: -0.05.

Companion Endangered

"Kael is about to take a killing blow. You can intervene but it leaves you exposed."

Save companion: they live, you take automatic injury. Self-preservation: companion makes their own death roll.

Player Near Death

"You're losing badly. Blood in your eyes. Flee, surrender, or fight to the death."

Flee: agility check. Surrender: consequences depend on enemy. Fight: final death roll with small willpower bonus.

Combat

Power Tiers

Can humans beat a dragon? Let's do the math.

Entity Tiers

Human: 1x
Exceptional warrior: 2.5x
Giant/troll: 6x
Dragon: 25x
Divine: 100x

Item Tiers

Common: 1x
Quality: 1.8x
Legendary: 4x
Mythic: 15x

Example: 12 Humans vs Dragon

Dragon CPR: 80 (base) * 25.0 (tier) = 2000

12 humans with legendary swords:
  Per soldier: 65 (trained) * 1.15 (long sword) * 4.0 (legendary) = 299
  Total: 299 * 12 = 3588
  Numbers bonus: 1 + log2(12) * 0.4 = 2.43x... but dragon is ONE target
  Effective: ~3588 * 1.0 (no numbers bonus against single target)

Ratio: 3588 / 2000 = 1.79 in humans' favor
With ±20% randomness: possible range 1.19x to 2.68x
Verdict: FAVORED but not guaranteed. Roll of the dice.

The math is transparent and feels right. Legendary gear matters. Numbers matter. Nothing is certain.

Social Systems

Persuasion

No persuasion stat. The player's actual words are the mechanic.

4-Dimension Evaluation

DimensionWeightWhat It Measures
Relevance35%Does this address what the NPC cares about?
Coherence25%Is the argument logically sound?
Tone Match25%Appropriate for this NPC's personality?
Info Validity15%Does the player actually know what they claim?

NPC Resistance

resistance = (stubbornness * 0.40 + willpower * 0.30
            + (100 - empathy) * 0.15 + faction_loyalty * 0.15) / 100

delta = (argument_strength - resistance + trust_bonus) * 20
// Typical range: -12 to +18 per exchange

Bluffing penalty: if you claim info you don't have and the NPC's perception beats your charisma, massive trust damage (-20).

Inertia at Extremes

Trust above 70: changes reduced by 40% (hard to push from "close" to "deep bond"). Trust below 20: changes reduced by 30% (hard to rebuild once broken).

Decay

Companions: -0.5/day if ignored. NPCs: -0.1/day naturally. High trust (60+) doesn't decay. Real relationships persist.

Some NPCs will never be interested. not_interested is a permanent ceiling. Others reject because they're grieving, committed, or you haven't earned enough trust. The game doesn't tell you why — the NPC just doesn't reciprocate.

Social Systems

Conversations as Currency

Some doors only open through knowledge.

The Guard's Brother

You learned in the tavern that the guard's brother died in the northern campaign. You say: "I heard about your brother. I'm sorry."

The interpreter checks: does the player actually have this knowledge (tracked in player.knowledge_log)? If yes: trust shift. If bluffing: suspicion.

The game tracks every piece of information you learn. Knowledge is a real resource — not just flavor text. The emotionally intelligent player has more doors open to them.

Player base stats: mean 55 (vs NPC mean 42). You're the protagonist, not a god.

Player Experience

Death & Consequences

Real stakes, telegraphed danger.

If you punch a bard in a crowded bar, maybe 15 dockworkers pile on you and you're dead. Game over. "Wow. That never happens in any game."

The Game Warns You

  • Narrator telegraphs danger: "The five men stop talking and stare. Their hands move under the table."
  • Companions voice concern: "I don't like this. We should leave."
  • Stat engine gives escape chances — submitting to bandits costs money but you live

Death Letters

When you die, the game generates an epilogue:

  • The war you were fighting — did your side win?
  • The companion you traveled with — where did they end up?
  • The merchant you cheated — did they recover?

Restart as New Character

Same world, years later. Your old character's actions are baked in. Your companion is now a bitter mercenary. Your name is a legend or a cautionary tale.

Player Experience

Ambient Life

The world doesn't wait for the player.

NPC-to-NPC Conversations

Sit in a tavern and do nothing. NPCs talk to each other. Not to you.

"...told him the bridge was out but he went anyway..." and then laughter.

You can: ignore it (scenery), listen closer (might get noticed), or interject (now you're part of it). Player perception determines how much you hear.

Philosophical Disagreement

Two scholars argue about whether the gods are real. A soldier and priest debate the war. The game tracks which positions you endorse. NPCs with matching worldviews trust you faster.

The Drunk Philosopher

Every world generates one: brilliant but ruined. Alcoholic ex-professor, disgraced court advisor. Sonnet-tier minimum. Rich context drawn from real philosophical traditions mapped onto world lore. The character players tell their friends about.

Silence as a Mechanic

"She looks at you for a long moment and says nothing."

Let the player sit in that. Most games fill every second. Silence is the most human thing a game can do.

Context & Memory

Three Memory Layers

Layer 1 — World State (always current JSON)

~500-800 tokens. Day, season, player location, health, hunger, faction standings, reputation. Updated by Python every turn. Never summarized.

Layer 2 — Player Action Memory (rolling, summarized)

~500-800 tokens. Recent turns: verbatim. Medium-term (2-3 days): compressed. Long-term: key moments only. Flagged moments (first kill, companion joined, major betrayal): never compressed. Summarizer runs every ~10 turns (Gemini Flash).

Layer 3 — Relationship Memory (per-NPC, sparse)

~100-300 tokens. Trust, attraction, flags, knowledge of player, last interaction summary. Injected only when talking to that NPC. Most NPCs have none. Companions have the most.

Context Assembly Per Call

System prompt                ~200-500 tokens
World state (Layer 1)        ~500-800 tokens
Player memory (Layer 2)      ~500-800 tokens
Scene context                ~200-400 tokens
NPC relationship (Layer 3)   ~100-300 tokens
Lore snippet (if triggered)  ~50-150 tokens
──────────────────────────────────────────
TOTAL PER CALL               ~1,500-3,000 tokens

Fits comfortably in any model's context window.

Context & Memory

Tagged Lore Retrieval

NPCs have knowledge tags. Lore is injected only when relevant.

How It Works

A character has tags: [educated, noble_family, academy_of_vael, skeptic].

When conversation topics match their tags, the system injects a small lore snippet (50-150 tokens) into their context for that specific response.

"The woman at the bar doesn't mention the philosopher until you say something that triggers it. Then the system pulls lore about that philosopher into her context, and she references it naturally. Knowledge feels organic, not performative."

Context Management Principles

  • Trim tool outputs — if a query returns 40 fields, pass only the 5 relevant ones
  • Structured fact extraction — pull key facts into persistent blocks, don't rely on summarization
  • Position-aware ordering — critical info at start and end (models miss the middle)
  • Summarize and reset when context gets stale in long sessions
